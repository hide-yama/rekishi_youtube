## システム概要

### 🎯 目的
フリーランス開業準備（2025年6月23日開始予定）のタスクを効率的に管理するシステムです。

### ⚡ 特徴
- **YAML一元管理**: `tasks.yml` でタスクの状態を集約
- **自動化**: CLIコマンドで日次タスクの抽出・同期
- **ID管理**: タスクを一意に識別（例：`LEGAL-001`）
- **双方向同期**: 日次ファイルの変更を自動でYAMLに反映
- **優先度・期限管理**: 視覚的にタスクの重要度を把握
- **🆕 重複チェック**: 新規タスク追加時の重複検出機能
- **🆕 柔軟な更新**: 期限・メモ・優先度の日次ファイル経由での変更
- **🔍 Obsidian対応**: YAMLデータを構造化Markdownに変換して視覚的管理

### 📁 ファイル構成
```
/開業準備_20250524/
├── package.json              ← Node.js設定
├── tasks.yml                 ← タスク一元管理（重要）
├── extract.js                ← 日次タスク抽出ツール
├── sync.js                   ← 双方向同期ツール（重複チェック・更新機能付き）
├── daily/                    ← 日次記録フォルダ
│   ├── 2025-05-24.md        ← 各日のタスク・振り返り
│   └── template.md          ← テンプレートファイル
├── docs/                     ← 参考資料
│   ├── 法的手続き/
│   ├── 税務資料/
│   ├── ブランディング/
│   └── 営業資料/
└── 使用マニュアル.md         ← 本ファイル
```

## 基本的な使い方

### 🌅 朝の準備（推奨時間：9:00）

```bash
# 今日のタスクファイルを生成
npm run today

# 生成されたファイルを確認
open daily/$(date +%F).md  # macOS
# または
code daily/$(date +%F).md  # VS Code
```

### 📝 日中の作業
1. 生成された日次ファイルでタスクを確認
2. 作業完了したら `- [x]` にチェック
3. 作業中は `- [~]` または🔄マークを追加
4. 🆕 新規タスクは `NEW:` 記法で追加
5. 🆕 既存タスクの更新は `UPDATE:` 記法で変更
6. 振り返りセクションに進捗や気づきを記録

### 🌆 夕方の同期（推奨時間：18:00）

```bash
# タスクの状態をYAMLに同期（重複チェック・更新機能付き）
npm run sync-today

# 変更をGitで記録（推奨）
git add tasks.yml daily/
git commit -m "chore: sync tasks $(date +%F)"
git push
```

## 日次ワークフロー

### 📊 完全な1日の流れ

#### Step 1: 朝のタスク生成
```bash
# 今日期限のタスクを抽出
npm run today

# または手動で日付指定
node extract.js --date 2025-06-10 > daily/2025-06-10.md
```

#### Step 2: タスクの実行・記録
生成されたファイルで：
- `- [ ]` 未着手のタスクを確認
- 完了したら `- [x]` にチェック
- 作業中は `- [~]` に変更（または🔄追加）
- 🆕 新規タスクは `- [ ] NEW: TASK-ID タスクタイトル` で追加
- 🆕 既存タスクの更新は `- UPDATE: TASK-ID due:YYYY-MM-DD memo:メモ priority:high` で変更
- 振り返りセクションに記録

#### Step 3: 夕方の同期
```bash
# 日次ファイルの変更をtasks.ymlに反映
npm run sync-today

# - 完了: 2件
# - 作業中: 1件
# - 未着手: 3件
# - 更新されたタスク: 3件
# 🆕 新規タスク: 1件追加
# 🔄 タスク更新: 2件更新
```

#### Step 4: Git記録
```bash
# 変更をコミット
git add .
git commit -m "feat: complete daily tasks $(date +%F)"
git push
```

## コマンドリファレンス

### npm scripts（推奨）

| コマンド | 説明 | 実際の動作 |
|---------|------|----------|
| `npm run today` | 今日のタスクファイル生成 | `node extract.js --date $(date +%F) > daily/$(date +%F).md` |
| `npm run sync-today` | 今日の同期実行（重複チェック・更新機能付き） | `node sync.js --file daily/$(date +%F).md` |
| `npm run extract` | extract.jsを実行 | 引数は手動指定が必要 |
| `npm run sync` | sync.jsを実行 | 引数は手動指定が必要 |
| `npm run obsidian` | Obsidian用Markdownファイル生成 | `node obsidian-export.js` |
| `npm run obsidian-category` | 特定カテゴリのみ生成 | `node obsidian-export.js --category` |
| `npm run obsidian-status` | 特定ステータスのみ生成 | `node obsidian-export.js --status` |

### 直接コマンド

#### extract.js（タスク抽出）

##### 基本的な使用法
```bash
# 基本形式
node extract.js --date YYYY-MM-DD [オプション]
node extract.js --overdue [オプション]
```

##### 📅 期限ベースの抽出

```bash
# 指定日期限のタスクを生成
node extract.js --date 2025-06-10 > daily/2025-06-10.md

# 全タスクを確認（期限無関係）
node extract.js --date 2025-06-10 --all

# 過去期限のタスクを抽出（重要！）
node extract.js --overdue

# 期間指定での抽出
node extract.js --from 2025-06-01 --to 2025-06-15

# 画面に出力のみ（ファイル生成なし）
node extract.js --date 2025-06-10 --all
```

##### 🔍 条件指定での抽出

```bash
# カテゴリで絞り込み
node extract.js --category "法的手続き"
node extract.js --category "金融関係"

# 優先度で絞り込み
node extract.js --priority high      # 高優先度のみ
node extract.js --priority medium    # 中優先度のみ
node extract.js --priority low       # 低優先度のみ

# ステータスで絞り込み
node extract.js --status open        # 未着手のみ
node extract.js --status in_progress # 作業中のみ
node extract.js --status completed   # 完了のみ
node extract.js --status backlog     # 後回しのみ
```

##### 🔧 組み合わせ抽出（高度）

```bash
# 高優先度の法的手続きタスク
node extract.js --category "法的手続き" --priority high

# 過去期限の高優先度タスク
node extract.js --overdue --priority high

# 特定期間の特定カテゴリ
node extract.js --from 2025-06-10 --to 2025-06-20 --category "事業基盤整備"

# 作業中の全タスク（進捗確認用）
node extract.js --status in_progress --all

# 6月前半の高優先度タスク（計画立案用）
node extract.js --from 2025-06-01 --to 2025-06-15 --priority high
```

##### 📊 実用的な抽出例

```bash
# 週次レビュー用：過去期限の確認
node extract.js --overdue > weekly_overdue_check.md

# 月次計画用：来月のタスク一覧

# 緊急度チェック：高優先度タスクのみ
node extract.js --priority high --all > high_priority_tasks.md

# 法的手続きの進捗確認
node extract.js --category "法的手続き" --all > legal_progress.md

# 完了タスクの確認（振り返り用）
node extract.js --status completed --all > completed_tasks.md
```

#### sync.js（双方向同期）

```bash
# 例：特定日のファイルを同期
```

## 🆕 新機能：タスクの追加・更新

### 📝 新規タスクの追加

#### 基本記法
日次ファイルに以下の形式で記述：
```markdown
- [ ] NEW: TASK-ID タスクタイトル
```

#### 実際の使用例
```markdown
- [ ] NEW: MARKETING-010 ウェブサイトのSEO対策
- [ ] NEW: LEGAL-009 開業届の提出準備
- [ ] NEW: INSURANCE-007 国民年金の切替手続き
```

#### 🔍 重複チェック機能

**高精度チェック（90-100%）**：
- **ID完全重複**: 既存のタスクIDと100%一致する場合
- **タイトル完全一致**: 大文字小文字を無視して完全一致する場合

**中精度チェック（70-80%）**：
- **キーワードベース類似**: 共通キーワードが2個以上、または全キーワードの50%以上で類似判定

#### 重複検出時の表示例
```bash
⚠️ 重複の可能性があるタスクが見つかりました:

🔴 高精度重複 (90-100%):
- LEGAL-001: 開業届の提出 (ID重複: 100%)

🟡 中精度重複 (70-80%):
- MARKETING-003: ポートフォリオ資料の準備 (類似度: 75%)

新規タスクの追加をスキップしました。
```

### 🔄 既存タスクの更新

#### 基本記法
日次ファイルに以下の形式で記述：
```markdown
- UPDATE: TASK-ID [更新項目]
```

#### 更新可能項目

| 項目 | 記法 | 例 |
|------|------|---|
| 期限変更 | `due:YYYY-MM-DD` | `due:2025-05-30` |
| メモ更新 | `memo:テキスト内容` | `memo:デイキャンプのため1日延期` |
| 優先度変更 | `priority:high\|medium\|low` | `priority:high` |

#### 実際の使用例

##### 単一項目の更新
```markdown
- UPDATE: INSURANCE-002 due:2025-05-28
- UPDATE: ENVIRONMENT-003 priority:high
- UPDATE: LEGAL-001 memo:失業保険との兼ね合いで6月23日以降に提出
```

##### 複数項目の同時更新
```markdown
- UPDATE: INSURANCE-002 due:2025-05-28 memo:デイキャンプのため1日延期 priority:high
- UPDATE: ENVIRONMENT-003 priority:high memo:docsファイル作成が急務
- UPDATE: INSURANCE-004 due:2025-06-01 priority:medium memo:ふるさと納税上限額の試算が必要
```

#### 更新結果の表示例
```bash
🔄 タスク更新:
- INSURANCE-002: 期限を2025-05-28に変更、メモを更新、優先度をhighに変更
- ENVIRONMENT-003: 優先度をhighに変更、メモを更新

```

## 🔍 Obsidian対応機能

### 📖 概要
`tasks.yml`（YAML形式）をObsidianで視覚的に確認するため、構造化されたMarkdownファイルに自動変換する機能です。

### 🤔 YAMLファイルとは？
**YAML**（ヤムル）は「YAML Ain't Markup Language」の略で：

#### 特徴
- **人間が読みやすい**データ構造化フォーマット
- **インデント**でデータの階層を表現
- **設定ファイル**やデータ交換によく使用
- **プログラムで処理しやすい**構造

#### YAML例（tasks.ymlの一部）
```yaml
- id: LEGAL-001
  title: 開業届の提出
  status: in_progress
  due: 2025-06-15
  category: 法的手続き
  priority: high
  source: docs/法的手続き/開業届.md
  memo: 失業保険との兼ね合いで6月23日以降に提出
```

#### 問題点
- **Obsidianで直接確認不可**: YAMLファイルはObsidianで構造化表示されない
- **視覚的把握困難**: データの関連性や全体像が見えにくい
- **検索・フィルタ制限**: Obsidianの豊富な機能を活用できない

### 🔧 解決策：obsidian-export.js

#### 基本的な使用方法
```bash
# 全カテゴリのMarkdownファイルを生成
npm run obsidian

# 出力先: obsidian-export/ ディレクトリ
# 生成ファイル数: カテゴリ数 + 2件（サマリー・期限別）
```

#### 高度な使用方法
```bash
# 特定カテゴリのみ生成
node obsidian-export.js --category=法的手続き

# 特定ステータスのみ生成
node obsidian-export.js --status=in_progress

# 出力先を指定
node obsidian-export.js --output=my-obsidian-vault

# 複数条件の組み合わせ
node obsidian-export.js --category=法的手続き --status=open --output=legal-tasks
```

### 📁 生成されるファイル構成

#### 1. 📄 00_タスクサマリー.md
**全体の統括ファイル**
- **統計情報**: 全タスク数、完了率、ステータス別内訳
- **カテゴリ別サマリー**: 各カテゴリの進捗状況
- **期限が近いタスク**: 上位10件の緊急タスク一覧
- **ナビゲーションリンク**: `[[カテゴリ名]]` でカテゴリ間移動

#### 2. 📁 カテゴリ別ファイル
**例：法的手続き.md、税務会計準備.md 等**
- **進捗状況**: カテゴリ内の統計情報
- **タスク詳細**: ステータス・優先度・期限・メモ
- **ソート機能**: 作業中 → 未着手 → 後回し → 完了
- **視覚的アイコン**: 🔄作業中、⭕未着手、✅完了、🔴高優先度

#### 3. 📅 期限別タスク一覧.md
**期限を軸とした横断ビュー**
- **期限順ソート**: 緊急度の高い順
- **視覚的アラート**: 🚨今日、⚠️経過日数表示
- **カテゴリ横断**: 期限を軸とした全体ビュー

### 🎯 Obsidianでの活用方法

#### 1. 基本セットアップ
```bash

# Step 2: Obsidianで obsidian-export/ フォルダを開く
```

#### 2. ナビゲーション機能

### 💡 活用のメリット

- **バックアップ**: Markdownファイルとして保存・共有

## カスタマイズ方法

### 📝 新しいタスクの追加

#### 1. 日次ファイル経由（推奨）
```markdown
- [ ] NEW: MARKETING-011 SNSアカウントの開設
```

#### 2. tasks.yml に直接追加
```yaml
# 新しいカテゴリの例
- id: MARKETING-011
  title: SNSアカウントの開設
  status: open
  due: 2025-06-18
  category: マーケティング
  priority: medium
  source: docs/マーケティング/SNS.md
```

### 🔄 既存タスクの変更

#### 1. 日次ファイル経由（推奨）
```markdown
- UPDATE: LEGAL-001 due:2025-06-12 priority:high memo:期限を前倒し
```

#### 2. tasks.yml で直接編集
```yaml
# 期限を変更
- id: LEGAL-001
  title: 開業届の提出
  due: 2025-06-12  # 変更前：2025-06-15
  priority: high   # 変更前：medium
  memo: 期限を前倒し  # 新規追加
```

### 🗂️ 新しいカテゴリの追加

1. 新しいプレフィックスを決定（例：`MARKETING-`）
2. tasks.ymlに該当タスクを追加
3. sync.jsの正規表現に追加（必要に応じて）

```javascript
// sync.js の該当部分を更新
const doneMatch = line.match(/^\s*-\s\[x\]\s+(TASK-\d+|LEGAL-\d+|ACCOUNTING-\d+|ENVIRONMENT-\d+|INSURANCE-\d+|FINANCE-\d+|CONTRACT-\d+|BRANDING-\d+|MARKETING-\d+|REVENUE-\d+|OPERATION-\d+|RISK-\d+)/);
```

### 📅 期限の一括調整

期限を一括で調整したい場合：

```bash
# tasks.yml をバックアップ
cp tasks.yml tasks.yml.backup

# テキストエディタで一括置換
# 例：2025-06-15 → 2025-06-20
```

## トラブルシューティング

### ❌ よくあるエラー

#### 1. `ファイルが見つかりません`
```bash
# エラー例
node sync.js --file daily/2025-06-10.md
# エラー: ファイルが見つかりません: daily/2025-06-10.md

# 解決方法
ls daily/  # ファイル存在確認
node extract.js --date 2025-06-10 > daily/2025-06-10.md  # ファイル生成
```

#### 2. `npm command not found`
```bash
# 解決方法
npm install  # 依存関係の再インストール
```

#### 3. `Permission denied`
```bash
# 解決方法
chmod +x extract.js sync.js  # 実行権限の付与
```

#### 4. `YAML syntax error`
```bash
# tasks.yml の構文エラー
# 解決方法：インデントとコロンの確認
# - id: LEGAL-001
#   title: タスク名  # ← コロンの後にスペース必須
```

#### 5. 🆕 `重複チェックエラー`
```bash
# 重複チェック機能でエラーが発生した場合
# 解決方法：タスクIDやタイトルの重複を確認
# 既存タスクと異なるIDまたはタイトルを使用
```

#### 6. 🆕 `UPDATE記法エラー`
```bash
# UPDATE記法の構文エラー
# 正しい記法：
- UPDATE: TASK-ID due:2025-05-30  # ✅ 正しい
- UPDATE TASK-ID due:2025-05-30   # ❌ コロンなし
- UPDATE: TASK-ID due=2025-05-30  # ❌ イコール使用
```
