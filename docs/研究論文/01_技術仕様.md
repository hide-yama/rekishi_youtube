### データフロー
1. **入力**: 日次Markdownファイルでのチェックボックス操作・ADD/UPDATE記法
2. **処理**: sync.jsによる解析・重複チェック・データ変換
3. **保存**: tasks.ymlへの構造化データ保存
4. **出力**: extract.js/obsidian-export.jsによる各種形式での表示生成
5. **バックアップ**: Gitによる版本管理

## データ構造

### YAML形式（tasks.yml）
```yaml
- id: TASK-001
  title: タスクタイトル
  category: legal
  status: open|in_progress|done|completed
  priority: high|medium|low
  due: "2025-MM-DD"
  memo: "メモ内容"
  reference: "docs/参照先.md"
```

### 日次ファイル形式
```markdown
# 2025-MM-DD のタスク一覧

## カテゴリ名
- [ ] TASK-001 タスクタイトル 🔴🔄
  - 期限: YYYY-MM-DD
  - 参照: docs/参照先.md

## 新規タスク追加
- ADD: TASK-002 新しいタスク priority:medium due:2025-MM-DD

## タスク更新
- UPDATE: TASK-001 due:2025-MM-DD memo:新しいメモ priority:high
```

## 技術的課題と解決策

### 1. 文脈継続性の実現
**課題**: AI対話の文脈を数日間保持
**解決策**:

- YAMLを単一真実源（Single Source of Truth）として設定

- Git版本管理による変更履歴保持

- モジュラー設計による機能分離
- 正規表現パターンによる柔軟な解析
- 段階的機能追加による影響範囲限定

```mermaid
sequenceDiagram
    participant User
    participant DailyMD as 日次ファイル(Markdown)
    participant SyncJS as sync.js
    participant TasksYML as tasks.yml(YAML)
    participant ExtractJS as extract.js
    participant CalendarScheduler as calendar-scheduler.js
    participant CalendarMD as calendar-output.md
    participant ObsidianExport as obsidian-export.js
    participant ObsidianMD as obsidian-export/*.md

    User->>DailyMD: タスク追加・更新・チェック
    DailyMD->>SyncJS: 変更検知・同期要求
    SyncJS->>TasksYML: 追加・更新・重複チェック
    TasksYML-->>SyncJS: YAMLデータ反映
    SyncJS->>DailyMD: 逆同期（YAML→MD）
    User->>ExtractJS: 日次タスク抽出コマンド実行
    ExtractJS->>TasksYML: 指定日・条件でタスク抽出
    ExtractJS->>DailyMD: Markdown形式で出力
    User->>CalendarScheduler: カレンダー割当コマンド実行
    CalendarScheduler->>TasksYML: 全タスク取得
    CalendarScheduler->>CalendarMD: 日付ごとに割当・出力
    User->>ObsidianExport: Obsidian用出力コマンド実行
    ObsidianExport->>TasksYML: 全タスク取得
    ObsidianExport->>ObsidianMD: カテゴリ・期限別にMD生成
```
