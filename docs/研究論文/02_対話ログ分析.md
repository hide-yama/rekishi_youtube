##### 策定されたルール体系

**基本原則**:
- タスクの新規追加・更新は原則としてデイリーファイル経由
- tasks.yml直接編集は緊急時のみ、必ずデイリーファイルにも反映
- 大量更新・複数タスク変更時は特にデイリーファイル経由を必須

**安全性の理由**:
- ファイル破損リスク回避、トレーサビリティ確保、設計思想準拠、復旧安全性

**実践的手順**:
```bash
# ✅ 安全：デイリーファイル経由
vim daily/$(date +%F).md  # 安全な変更
npm run sync-today        # 確実な同期

# ❌ 危険：tasks.yml直接編集（緊急時のみ）
vim tasks.yml  # ファイル破損リスク高
```
